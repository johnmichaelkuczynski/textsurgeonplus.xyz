REBUILD “BOOK TO DATABASE” FROM SCRATCH
Core Directive
The previous implementation failed. It treated almost every sentence as a “position.” That is unacceptable.
Book to Database must extract only genuine intellectual commitments. Everything else (exposition, questions, transitions, examples, rhetorical flourishes, pure description) must be ruthlessly discarded.
The goal is a clean, high-signal structured representation of a book’s actual doctrinal content — not a dump of nearly every sentence.

Absolute Rules for Position Extraction

A position is a substantive claim the author is committing to.
The following are never positions:
Questions
Imperatives (“Consider…”, “Let me explain…”)
Transitional sentences
Pure examples or illustrations
Sentence fragments
Rhetorical restatements
“The answer is clear”, “What’s the explanation?”, etc.

Prefer fewer, stronger positions over many weak ones.
For a short essay (~1800–2500 words), target 8–18 high-quality positions.
For a full book, target a disciplined set, not hundreds.

Every position must be rewritten as a clean, self-contained declarative claim in the author’s voice (or close paraphrase). Do not leave raw, messy fragments.
Classify strictly:
core = central doctrinal commitments
supporting = claims that directly back a core position
doctrinal = only use sparingly for explicit theoretical principles
Never mark weak material as core.

Confidence must be calibrated. Most real positions should land between 70–95. Reserve 100 for the strongest, clearest thesis statements.

Intelligence Scoring Rules (new, stricter)
The old metrics were inflated and soft. Replace them with these standards:

Claim Density: Count only genuine positions + high-value supporting claims. Divide by word count (per 1000 words).
Conceptual Compression: How much real intellectual content is packed tightly vs. padded.
Redundancy Score: Penalize repeated reformulations of the same point.
Filler Ratio: Proportion of text that is pure exposition, rhetoric, or low-information.
Overall Score: Weighted combination. A dense, high-signal philosophical essay should be able to score 80+, but most academic writing should land lower.
Qualitative assessment must be specific and critical, not bland.


Processing Rules

Use the existing outlineService for long documents. Do not invent a new chunking system.
Process section by section, then aggregate and aggressively deduplicate and filter.
After extraction, run a dedicated cleaning pass that:
Removes non-claims
Merges near-duplicates
Downgrades or deletes weak items
Forces clean declarative wording

Prefer Claude (Anthropic) for the extraction and cleaning stages when available. It is currently better at discrimination than most alternatives.
Quotes must be high-signal only. Aim for quality over quantity (roughly 8–20 strong quotes for a short essay).


Frontend & Persistence

Keep the Book to Database mode.
Results view must make the distinction between core and supporting positions visually obvious.
Allow Save / Load of the Book Database object.
Add a basic “Query this Book Database” capability that works against the cleaned positions + quotes (not the raw text).


Acceptance Criteria (non-negotiable)
On the Chomsky test essay (~1878 words), a successful run must produce approximately:

10–16 strong, clean positions (not 70+)
Clear separation of the two main contributions as core
High-signal quotes only
Intelligence score that feels earned, not inflated
Concept clusters that actually group related real claims
Almost zero rhetorical questions or transitional sentences appearing as positions

If the output still contains large numbers of non-claims, the implementation has failed.
